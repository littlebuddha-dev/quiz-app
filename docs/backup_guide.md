# 大規模データのバックアップ・復元ガイド

JSON形式のバックアップ・復元で「Unterminated string in JSON」や「Unexpected end of JSON input」といったエラーが出る場合、データサイズ（特に画像データ）がサーバーやブラウザの制限（一般的に10MB〜50MB程度）を超えている可能性があります。

このような大規模なデータを確実に移行・バックアップするには、SQLiteのデータベースファイルを直接コピーする方法が最も推奨されます。

## 推奨手順：データベースファイルの直接コピー

本アプリケーションはSQLiteを使用しているため、`prisma/dev.db` という1つのファイルをコピーするだけで、すべての設定・クイズ・履歴を完全にバックアップおよび復元できます。

### 1. リモートサーバーからのバックアップ（ダウンロード）

ターミナル（コマンドプロンプト）で以下のコマンドを実行し、サーバー上のデータベースを手元に保存します。

```bash
# SSHを利用してリモートからローカルへコピー
scp ユーザー名@サーバーのIPアドレス:/保存先パス/quiz-app/prisma/dev.db ./backup_dev.db
```

※パスは、実際にアプリをインストールした場所に合わせて変更してください。

### 2. ローカルサーバーへの復元（アップロード）

受け取った `backup_dev.db` を、ローカル環境の `prisma/` ディレクトリに `dev.db` として上書き配置します。

```bash
# 既存のファイルをバックアップ（念のため）
mv prisma/dev.db prisma/dev.db.bak

# バックアップファイルを配置
cp path/to/backup_dev.db prisma/dev.db
```

### 3. 注意事項
- **上書きの確認**: 復元を行うと、移動先の既存データはすべて消去されます。必ず事前にバックアップを取ってください。
- **実行環境**: ファイルを上書きする際は、Node.js（Next.js）のプロセスを一度停止してから行うとより安全です。
- **ファイルパーミッション**: サーバーへアップロードした後は、書き込み権限（644 または 666 など）が正しく設定されているか確認してください。

---

この方法はJSONパースのオーバーヘッドやサイズ制限がないため、数万件のクイズデータがあっても数秒で確実に復元が可能です。
